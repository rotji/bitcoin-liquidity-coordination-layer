# Bitcoin Liquidity Network — Project Introduction

## Vision

To build the foundational infrastructure for Bitcoin-secured DeFi by coordinating, routing, and optimizing sBTC liquidity across the Stacks ecosystem. The project aims to solve liquidity fragmentation, improve capital efficiency, and provide a protocol-agnostic base layer for all DeFi protocols, wallets, and applications on Stacks.

## Context

Bitcoin DeFi is in its infancy. sBTC unlocks Bitcoin capital for smart contracts, but liquidity is thin and fragmented across DEXs, lending, and vaults. Without a coordination layer, capital remains idle, slippage is high, and developer UX suffers. The Bitcoin Liquidity Network (BLN) is designed to address these pain points by providing a unified liquidity abstraction and routing engine.

## Key Features

- **Liquidity Indexer:** Aggregates sBTC pools, lending, and vaults across protocols.
- **Routing Engine:** Finds optimal routes for swaps, lending, and capital allocation.
- **SDK & API:** Easy integration for wallets, protocols, and frontends.
- **Composable Middleware:** Designed to be the base layer for all Bitcoin DeFi apps.

## Why Now?

- sBTC is launching, creating new opportunities for Bitcoin capital to flow into DeFi.
- Stacks ecosystem is early-stage, so infrastructure can be built before fragmentation becomes unmanageable.
- No existing solution provides unified liquidity routing for Bitcoin DeFi.

## Target Users

- DeFi protocols (DEXs, lending, vaults)
- Wallets and aggregators
- Developers building on Stacks
- Power users seeking best execution

## Grant Alignment

The project is ecosystem-first, infrastructure-focused, and designed for maximum composability. It aligns with the goals of the Stacks Foundation and Bitcoin DeFi growth.

## Next Steps

1. Finalize system architecture and technical design
2. Build MVP: indexer, router, API, SDK
3. Integrate with 2–3 major protocols
4. Launch analytics dashboard
5. Expand to lending/vault routing post-MVP
