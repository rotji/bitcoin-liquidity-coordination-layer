# System Design Philosophy

This document outlines the core principles and philosophies guiding the system design of the Bitcoin Liquidity Network project.

---

## 1. Infrastructure-First

- Prioritize building foundational layers that enable ecosystem growth.
- Avoid over-engineering; focus on composability and extensibility.

## 2. Protocol-Agnosticism

- Design interfaces and abstractions that work across DEXs, lending, and vaults.
- Avoid protocol lock-in or favoritism.

## 3. Security by Default

- Minimize attack surface by keeping contracts minimal.
- Use open-source, audited components where possible.

## 4. Developer Experience

- Provide clear APIs, SDKs, and documentation.
- Make integration easy for wallets, protocols, and frontends.

## 5. Open Source & Community

- All core components are open source.
- Encourage contributions and ecosystem alignment.

## 6. Iterative Delivery

- Build MVPs, gather feedback, and iterate quickly.
- Avoid scope creep in early phases.

---

_This philosophy should guide all major architectural and technical decisions in the project._
